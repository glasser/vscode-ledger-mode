Reconcile File Editor Test Cases

What This Shows

Tests for the file editor that updates reconciliation status markers in ledger files.

Features tested:

  • Updating posting statuses (uncleared → pending → cleared)
  • Smart normalization (using transaction headers when all postings have same status)
  • Precondition validation (checking expected current status before updating)
  • Error handling for invalid line numbers and edge cases
Legend: Target posting lines | Changed lines
